# 情侣模块功能设计

## 目标与原则
- **目标**：为情侣提供轻量、私密且有仪式感的互动空间，同时支持公共分享与纪念。
- **原则**：
  - 默认隐私优先（可选公开/仅双方可见/密码访问）。
  - 数据结构可扩展，避免一次性大而全。
  - 互动与纪念分层，便于逐步迭代。

## 关系管理
1. **发送/接受情侣邀请**：支持通过用户名或邀请链接发起邀请。
2. **双向确认机制**：双方确认后关系生效（含过期与撤销逻辑）。
3. **解除关系**：支持解除关系并保留历史数据（可选归档）。
4. **关系状态提示**：在个人中心与情侣页展示状态（待确认/已绑定/已解除）。

## 互动功能
1. **情侣相册**：
   - 分组相册（旅行、日常、纪念日）。
   - 支持批量上传、封面图与描述。
2. **共享日历**：
   - 共享日程与可见性（私人/共享）。
   - 支持提醒与重复事件。
3. **共同待办事项**：
   - 支持分配与完成状态。
   - 支持“轮值”或“共同完成”。
4. **留言板**：
   - 类似便签/留言墙，支持置顶与情感标签。

## 纪念功能
1. **时间轴**：
   - 恋爱天数计数器与相识/恋爱/结婚起始日期。
2. **重要日期管理**：
   - 纪念日、生日、节日等。可设置提醒时间。
3. **里程碑记忆**：
   - 关键事件（求婚、旅行、搬家）图文记录。

## 互动提醒
1. **节日倒计时提醒**：支持提前 X 天提醒。
2. **生日倒计时提醒**：支持发送站内/邮件提醒。

## 可扩展的实用功能（建议）
- **情侣任务/打卡**：例如“每日一句话”“一周约会计划”。
- **回忆卡片**：随机展示历史图片或留言。
- **共享心愿单**：纪念日礼物或旅行计划清单。
- **访客统计**：记录访问次数与最近访问时间（可选隐私开关）。

## 权限与可见性建议
- 私密等级：公开 / 仅双方 / 密码访问 / 仅本人。
- 基础审计：敏感操作记录（绑定/解绑/删除）。

## 分阶段落地建议
1. **MVP**：关系管理 + 情侣主页 + 相册 + 留言板。
2. **V1**：时间轴 + 重要日期提醒 + 共享日历。
3. **V2**：共同待办 + 任务打卡 + 访客统计。
